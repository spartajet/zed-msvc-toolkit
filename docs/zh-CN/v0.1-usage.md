# V0.2 使用说明

## 适用范围

V0.2 覆盖 Windows + Visual Studio 2022+ + MSVC 项目的 clangd 智能感知路径，并支持 CMake 编译数据库探测。

V0.2 不实现：

- CMake configure/build 命令
- DAP 调试
- `vsdbg.exe` 下载
- 已有 `.clangd` 的 YAML 合并

## 依赖

- Windows 10/11
- Visual Studio 2022 或更新版本
- "使用 C++ 的桌面开发"工作负载
- MSVC v143+ build tools
- Windows SDK
- PATH 中可找到 `clangd.exe`

## `.clangd` 策略

V0.2 采用保守策略：

- 工作区没有 `.clangd` 时，扩展会探测 MSVC/Windows SDK 并生成 MSVC 兼容配置内容。
- **V0.2 新增：** 自动探测 `compile_commands.json`（在根目录或 `build/` 子目录），检测到时使用 `CompilationDatabase` 配置。
- 受当前 Zed extension API 限制，扩展不能直接写入工作区文件；缺少 `.clangd` 时会返回明确提示，要求用户手动创建 `.clangd` 并粘贴生成内容。
- 工作区已有 `.clangd` 时，扩展不覆盖用户配置。
- Windows SDK 探测失败时，生成内容会包含手动补充 SDK include 的注释。

## 编译数据库优先级

当检测到 `compile_commands.json` 时：

1. 根目录的数据库优先于 `build/` 子目录
2. 生成的 `.clangd` 将包含 `CompilationDatabase: <路径>` 配置
3. clangd 将优先使用编译数据库中的编译命令和 include 路径
4. MSVC/SDK include 路径仍会添加作为备用

## clangd 参数

V0.2 启动 clangd 时会加入：

```text
--header-insertion=never
```
