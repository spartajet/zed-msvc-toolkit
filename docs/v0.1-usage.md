# V0.2 Usage Guide

## Scope

V0.2 covers clangd intelligence path for Windows + Visual Studio 2022+ + MSVC projects, and supports CMake compile database detection.

V0.2 does not implement:

- CMake configure/build commands
- DAP debugging
- `vsdbg.exe` download
- YAML merge with existing `.clangd`

## Dependencies

- Windows 10/11
- Visual Studio 2022 or newer
- "Desktop development with C++" workload
- MSVC v143+ build tools
- Windows SDK
- `clangd.exe` available in PATH

## `.clangd` Strategy

V0.2 adopts a conservative strategy:

- When workspace has no `.clangd`, extension detects MSVC/Windows SDK and generates MSVC-compatible configuration content.
- **V0.2 Addition:** Automatically detects `compile_commands.json` (in root or `build/` subdirectory), uses `CompilationDatabase` configuration when detected.
- Due to current Zed extension API limitations, extension cannot write directly to workspace files; when `.clangd` is missing, returns clear prompt asking user to manually create `.clangd` and paste generated content.
- When workspace already has `.clangd`, extension does not overwrite user configuration.
- When Windows SDK detection fails, generated content includes comments for manually supplementing SDK includes.

## Compile Database Priority

When `compile_commands.json` is detected:

1. Root directory database takes priority over `build/` subdirectory
2. Generated `.clangd` will include `CompilationDatabase: <path>` configuration
3. clangd will prioritize compile commands and include paths from compile database
4. MSVC/SDK include paths still added as fallback

## clangd Arguments

V0.2 starts clangd with:

```text
--header-insertion=never
```
